1. 关于imagemagick的错误

org-compile-file: File "/tmp/orgtexI6Eqwl.pdf" wasn’t produced.  Please adjust ‘imagemagick’ part of ‘org-preview-latex-process-alist’.

1.1. 错误1

参考链接： https://stackoverflow.com/questions/52998331/imagemagick-security-policy-pdf-blocking-conversion 此错误可能是由于 ImageMagick 的安全策略导致，解决方法：

  • 首先确保 Ghostscript ≥9.24， gc --version

  • 在 /etc/ImageMagick-6/policy.xml 文件中注释以下内容 

    <!-- disable ghostscript format types -->
<!--policy domain="coder" rights="none" pattern="PS" /-->
<!--policy domain="coder" rights="none" pattern="PS2" /-->
<!--policy domain="coder" rights="none" pattern="PS3" /-->
<!--policy domain="coder" rights="none" pattern="EPS" /-->
<!--policy domain="coder" rights="none" pattern="PDF" /-->
<!--policy domain="coder" rights="none" pattern="XPS" /-->

1.2. 错误2

参考链接： https://emacs.stackexchange.com/questions/31408/error-message-when-previewing-latex-snippet-in-org-file

也有可能时缺少 ulem.spy 包导致，确认是否安装 texlive-plain-generic 包， apt install texlive-plain-generic

具体错误详情可以检查 /tmp/orgtexXXXXXX.log /tmp/orgtexXXXXXX.tex 。
