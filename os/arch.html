1 安装

1.1 usb制作启动盘

首先需要将整个的U盘内容备份，然后依次执行如下命令

删除U盘上所有的分区信息: 

$ sudo wipefs --all /dev/sdb

将iso写入到U盘: 

$ sudo dd if=archlinux-2018.12.01-x86_64.iso of=/dev/sdb bs=8M status=progress oflag=sync

1.2 查看BIOS设置，检查引导方式：MBR(legacy)/GPT(UEFI)

这里需要区分对待两种不同的引导方式：

MBR 方式
只需要一个磁盘分区作为系统分区即可，剩余分区作为独立分区单独挂在到系统中，可以不加 swap分区，后面可以在通过命令方式添加交换文件。
GPT 方式
需要额外多一个分区作为efi引导用 主要因为有时候通过efi引导的系统键盘有时候不能用

1.3 安装系统

1.3.1 分区

BIOS 与 MBR

挂载点 分区 分区类型 建议大小
[SWAP] /dev/swap_partition（交换空间分区） Linux swap (交换空间) 大于 512 MiB
/mnt /dev/root_partition（根分区） Linux 剩余空间

UEFI 与 GPT

挂载点 分区 分区类型 建议大小
       
/mnt/boot 或 /mnt/efi /dev/efi_system_partition（efi 系统分区） EFI 系统分区 至少 260 MiB
[SWAP] /dev/swap_partition（交换空间分区） Linux swap (交换空间) 大于 512 MiB
/mnt /dev/root_partition（根分区） Linux x86-64 根目录 (/) 剩余空间

1.3.2 格式化磁盘：

$ cfdisk /dev/sdx

MBR: 

$ mkfs.ext4 /dev/sdx

GPT: 

$ mkfs.fat /dev/sdx1
$ mkfs.ext4 /dev/sdx2

1.3.3 挂载分区

MBR 

# mount /dev/sdx /mnt

GPT 

$ mount /dev/sdx2 /mnt/
$ mkdir /mnt/boot/efi
$ mount /dev/sdx1 /mnt/boot/efi

如果创建了swap，使用以下命令启用： 

$ swapon /dev/swap_partition（交换空间分区）

1.3.4 安装

# pacstrap /mnt base linux linux-firmware
  或者
# pacstrap /mnt base linux-lts linux-firmware

base 软件包并没有包含 Live 环境中的全部程序。因此要获得一个功能齐全的基本系统，可能需要安装其他软件包。特别要考虑安装：

  • 管理所用文件系统的用户工具 xfs btrfs 需要安装相应的管理工具；
  • 访问 RAID 或 LVM 分区的工具；
  • 未包含在 linux-firmware 中的额外固件；
  • 联网 所需要的程序(dhcpd)；
  • 文本编辑器 比如 nano vim；
  • 访问 man 和 info 页面的工具：man-db, man-pages 和 texinfo。
  • 要安装其他软件包或软件包组 (比如 base-devel)，请将它们的名字追加到上文的 pacstrap 命令后 (用空格分隔)，或者也可以在 Chroot 进新系统后使用 pacman 手动安装软件包或软件包组。packages.x86_64 中可以看到不同软件包或软件包组间的差异。

1.3.5 配置系统

Fstab 用以下命令生成 fstab 文件 (用 -U 或 -L 选项设置UUID 或卷标)： 

# genfstab -U /mnt >> /mnt/etc/fstab

强烈建议在执行完以上命令后，后检查一下生成的 /mnt/etc/fstab 文件是否正确。

1.3.6 Chroot

Change root 到新安装的系统： 

# arch-chroot /mnt

1.3.7 时区

设置时区： 

# ln -sf /usr/share/zoneinfo/Asia/Shanghai /etc/localtime

运行 hwclock(8) 以生成 /etc/adjtime： 

# hwclock --systohc

这个命令假定硬件时间已经被设置为 UTC 时间。

1.3.8 本地化

本地化的程序与库若要本地化文本，都依赖 Locale，后者明确规定地域、货币、时区日期的格式、字符排列方式和其他本地化标准。

需在这两个文件设置：locale.gen 与 locale.conf。

编辑/etc/locale.gen 然后移除需要的 地区 前的注释符号 #。

接着执行 locale-gen 以生成 locale 信息： 

# locale-gen

然后创建 locale.conf 文件，并 编辑设定 LANG 变量，比如： 

# echo "LANG=en_US.UTF-8" > /etc/locale.conf

另外对于中文用户： 

用户可以设置自己的 locale，详情请参阅 Locale#Overriding system locale per user session 或 Locale (简体中文)#设置 locale；

将系统 locale 设置为 en_US.UTF-8 ，系统的 log 就会用英文显示，这样更容易判断和处理问题；
也可以设置为 en_GB.UTF-8 或 en_SG.UTF-8，附带以下优点:
进入桌面环境后以 24 小时制显示时间；
LibreOffice 等办公软件的纸张尺寸会默认为 A4 而非 Letter(US)；
可尽量避免不必要且可能造成处理麻烦的英制单位。

1.3.9 网络配置

创建 hostname 文件: 

# echo "myhostname" > /etc/hostname

添加对应的信息到 hosts: 

# cat  /etc/hosts
127.0.0.1   localhost
::1     localhost
127.0.1.1   myhostname.localdomain  myhostname

如果系统有一个永久的 IP 地址，请使用这个永久的 IP 地址而不是 127.0.1.1。

1.3.10 Root 密码

设置 Root 密码： 

# passwd

1.3.11 安装引导程序(UEFI)

首先安装软件包 grub 和 efibootmgr。其中“GRUB”是启动引导器，“efibootmgr”被 GRUB 脚本用来将启动项写入 NVRAM。 

# pacman -S grub efibootmgr

esp替换为EFI分区挂载目录 

# grub-install --target=x86_64-efi --efi-directory=esp --bootloader-id=GRUB

在配置完成后，生成主配置文件。 

# grub-mkconfig -o /boot/grub/grub.cfg

1.3.12 退出重启系统

# exit
# umount /mnt -R reboot
