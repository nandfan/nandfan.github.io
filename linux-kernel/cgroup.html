1 cgroup介绍

1.1 概述

CGroup 是 Control Groups 的缩写，是 Linux 内核提供的一种可以限制、记录、隔离进程组 (process groups) 所使用的物力资源 (如 cpu memory i/o 等等) 的机制。

1.2 功能

  • Resource limitation: 限制进程组可以使用的资源数量。 比如 磁盘、内存、网络等。
  • Prioritization: 进程组的优先级控制。
  • Accounting: 记录进程组使用的资源。
  • Isolation: 进程组隔离。
  • Control: 进程组控制。

1.3 基础概念

cgroup是linux下的一种将进程按照组进行管理的机制， 从用户层面来看，cgroup将系统中的所有进程组织成一棵或者多棵独立的树，每棵树都包含系统中所有的进程，树的节点是进程组， 每棵树可以和一个或者多个subsystem关联， 树的作用是将进程分组， subsystem作用于进程组。

  • subsystem 子系统作为资源控制器，用来调度资源或者控制资源的使用上限， 可以是要作用于进程组的任意对象，例如虚拟化子系统。
  • hierarchy 一个hierarchy可以理解为一棵cgroup树， 树的每个节点为一个进程组， 每棵树会与零到多个subsystem关联。 一棵树中会包含linux系统中的所有进程， 每个进程在一棵树中只能属于一个节点（进程组）。 系统中可以有多棵cgroup树，即一个进程可以属于多个进程组，进程组和不同的subsystem相关联。 目前linux支持12种subsystem， 因此最多支持12棵cgroup树， 每棵树关联一个subsystem。 当一棵树不和任何subsystem关联时， cgroup树只是将进程分组，有应用程序决定在分组之上做什么， systemd就是这样的例子。

2 cgroup使用

cgroup相关的操作都是基于内核中的cgroup virtual filesystem， 使用cgroup很简单， 挂载这个文件系统就可以。一般挂载到/sys/fs/cgroup目录下。

下面用到的xxx可以为任意字符串， 取一个有意义的名称就可以， 使用mount查看时， xxx会显示在第一列。

2.1 挂载

在很多使用systemd的系统中， systemd已经帮我们将各个subsystem和cgroup树关联并挂在好了

  • 挂载一棵和所有subsystem关联的cgroup树到/sys/fs/cgroup
mount -t cgroup xxx /sys/fs/cgroup
  • 挂载一棵和cpuset subsystem关联的cgroup树到 /sys/fs/cgroup/cpuset
mkdir  /sys/fs/cgroup/cpuset
mount -t cgroup -o cpuset xxx /sys/fs/cgroup/cpuset
  • 挂载一棵与cpu和cpuacct sybsystem关联的cgroup树到/sys/fs/cgroup/cpu,cpuacct
mkdir /sys/fs/cgroup/cpu,cpuacct
mount -t cgroup -o cpu,cpuacct xxx /sys/fs/cgroup/cpu,cpuacct
  • 挂载一棵与cgroup树， 但不关联任何subsystem， 下面就是systemd所用到的方式
mkdir /sys/fs/cgroup/systemd
mount -t cgroup -o none,name=systemd xxx /sys/fs/cgroup/systemd

创建并挂载好一棵cgroup树之后， 就有了树的根节点， 即根cgroup，这时候就可以通过创建文件夹的方式创建子cgroup， 然后再往每个子cgroup中添加进程。

2.2 hierarchy和subsystem关联规则

  1. 同一个hierarchy能够附加一个或多个subsystem。
  2. 一个subsystem可以附加到多个hierarchy， 当且仅当这些hierarchy只关联此subsystem。
  3. 系统每次新建一个hierarchy时， 该系统上的所有task默认构成了这个新建的hierarchy的初始化cgroup， 这个cgroup也称为root cgroup。 在每个hierarchy中， task只能存在与其中一个cgroup中， 即一个task不能存在于同一个hierarchy中的不同cgroup中。 如果将task添加到同一个hierarchy中的另一个cgroup中， 则会从第一个cgorup中移除。
  4. 进程（task）在fork时创建的子任务（child task）默认与原task在同一个cgroup中， 但是child task允许被移动到不同的cgroup中。

2.6 cgroups挂载





cgroups文件系统挂载





cgroups一般挂载到/sys/fs/cgroup目录下， 也可以指定其他的挂载目录。





Linux中，用户可以使用mount命令挂载 cgroups 文件系统，格式为: mount -t cgroup -o subsystems name /cgroup/name，其中 subsystems 表示需要挂载的 cgroups 子系统， /cgroup/name 表示挂载点，如上文所提，这条命令同时在内核中创建了一个cgroups 层级结构。





比如挂载 cpuset, cpu, cpuacct, memory 4个subsystem到/cgroup/cpu_and_mem 目录下，就可以使用 mount -t cgroup -o remount,cpu,cpuset,memory cpu_and_mem /cgroup/cpu_and_mem
