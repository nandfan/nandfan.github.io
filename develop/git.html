2. 技巧

2.1. 显示文件历史提交内容

git log命令

1、git log – filename(git log filename) 可以看到该文件相关的commit记录

2、git log -p filename 可以显示该文件每次提交的diff

3、git show comit_id filename 可以查看某次提交中的某个文件变化

4、git show commit_id 查看某次提交

5、gitk –follow filename 以图形化界面的方式显示修改列表

2.2. 获取所有tag

$ git fetch  --tags

2.3. 强制回退版本（不留记录）

git reset --hard commitid
git push -f

2.4. 合并几个commit

git rebase -i  commitid


pick 的意思是要会执行这个 commit
squash 的意思是这个 commit 会被合并到前一个commit

2.5. 修改上一个commit

git commit --amend

2.6. github/gitlab 拉取PR/MR进行review

假设远程仓库为 upstream。

2.6.1. Github

#直接拉取
$ git pull upstream  pull/2618/head

#新建分支pr2618
$ git fetch upstream  pull/2618/head:pr2618

2.6.2. Gitlab

#直接拉取
$ git pull  upstream  refs/merge-requests/2618/head

#新建分支mr2618
$ git fetch upstream  refs/merge-requests/2809/head:mr-upstream-2809
