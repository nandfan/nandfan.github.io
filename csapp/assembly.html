2.3. 压入和弹出栈数据

遵循“后进先出”原则，push将数据压入栈，pop弹出栈数据

指令 效果 描述
pushq S R[%rsp] <- R[%rsp]-8; M[R[%rsp]] <- S 将四字压入栈
popq D D <- M[R[%rsp]]; R[%rsp] <- R[%rsp]+8 将四字弹出栈

压栈先将栈指针减8，之后将值写入新的栈顶地址，相当于一下操作： 

aubq $8,%rsp
movq %rbp,(%rsp)

出栈先将四字从栈顶读出，然后将栈指针加8。 

movq (%rsp),%rax
addq $8,%rsp

3.1. 加载有效地址

leaq是加载有效地址（load effective address）指令,相当于movq指令的变形。它的指令形式是从内存读数据到寄存器，但实际上根本没有内存引用。它的第一个操作数看上去是一个内存引用，但该指令并不是从指定的位置读入数据，而是将有效地址写入到目的操作数。例如，假设%rdx值为x， leaq 7(%rdx,%rdx,4),%rax ，该指令将寄存器 %rax 的值设置为 5x+7。

leaq示例： 

long scale(long x, long y, long z) {
        long t = x + 4 * y + 12 * z;
        return t;
}

scale:
  leaq (%rdi, %rsi, 4), %rax   // x + 4*y
  leaq (%rdx, %rdx, 2), %rdx   // z + 2*z = 3*z
  leaq (%rax, %rdx, 4), %rax   // (x+4*y) + 4*(3*z) = x+4y+12z
  ret

3.2. 一元和二元操作

第二组中的操作是一元操作，只有一个操作数。可以是寄存器，也可以是内存位置。

第三组是二元操作，第二个操作数即是源又是目的。

3.3. 位移操作

最后一组是位移操作，先给出位移量，然后是要位移的数。可以进行算术位移和逻辑右移。位移量可以是立即数，或者放在单字节寄存器 %cl 中（如果是寄存器，只能使用%cl）。x86-64中，位移操作对w位长的数据值进行操作，位移量由%cl寄存器的低m位决定的，这里的 2m=W。高位会被忽略。所以寄存器%cl值为0xFF时，指令salb位移7位，salw位移15位，sall位移31位，salq位移63位。

左移 SAL 和 SHL 效果相同，右边填0。右移指令 SAR 执行算术位移（填符号位），SHR执行逻辑位移（填0）。

4.1. 条件码

CPU维护一组单个位的条件码寄存器。描述了最近的算术或逻辑操作的属性。

CF 进位标志
最近的操作使最高位产生进位，可用来检查无符号操作的溢出
ZF 零标志
最近的操作得出的结果为0
SF 符号标志
最近的操作得到的结果为负数
OF 溢出标志
最近的操作导致一个补码溢出–正溢出或负溢出

假设用一条ADD指令完成等价于 C表达式 t=a+b 的功能，a、b、t都是整形，然后根据下面的C表达式来设置条件码：

CF (unsigned)t < (unsigned)a 无符号溢出
ZF (t==0) 零
SF (t<0) 负数
OF (a<0 = b<0) && (t<0 ! a<0) 有符号溢出

注意：leaq指令不改变任何条件码。前面表列出的其他整数运算操作会设置条件码。对于逻辑操作，进位和溢出标志设置为0。对于位移操作，进位表示设置为最后一个被溢出的位，溢出标志设置为0。INC和DEC会设置溢出和零标志，但不会改变进位标志。

下表中的指令，只设置条件码，不改变其他寄存器。例如CMP指令根据两个操作数之差设置条件码，除了只设置条件码，不更新目的寄存器外，CMP指令和SUB指令行为相同。TEST指令行为和AND相同，但是只设置条件码，不改变目的寄存器值。

Table 7: 比较和测试指令。这些指令不修改任何寄存器的值，之设置条件码
指令 基于 描述
CMP S1,S2 S2 - S1 比较
_{cmpb}   比较字节
_{cmpw}   比较字
_{cmpl}   比较双字
_{cmpq}   比较四字
TEST S1,S2 S1&S2 测试
_{testb}   测试字节
_{testw}   测试字
_{testl}   测试双字

Test典型用法：

  1. 两个操作相同。例如： testq %rax,%rax ，检查%rax是负数、零、正数。
  2. 一个操作数为掩码。 用来指示哪些位应该被测试。

4.2. 访问条件码

条件码通常不直接读取，常用的三种方法：

  1. 根据条件码的某种组合，将一个字节设置为0或者1
  2. 可以条件跳转到程序的某个其他部分
  3. 可以有条件的传送数据
  4. e: equal 相等
  5. l: less 小于
  6. g: greater 大于
  7. a: above 超过
  8. b: below 低于
Table 8: SET指令。每条指令根据条件码的某种组合，将一个字节设置为0或者1。
指令 同义名 效果 设置条件
sete D setz D <- ZF 相等/零
setne D setnz D <- ~ZF 不等/非零
sets D   D <- SF 负数
setns D   D <- ~SF 非负数
setg D setnle D <- ~(SF^OF)&~ZF 大于（有符号>）
setge D setnl D <- ~(SF^OF) 大于等于（有符号>=）

set指令的目的操作数是低位单字节寄存器元素之一，或是一个字节的内存位置，set指令将该字节设置成0或者1。

4.3. 跳转指令

下表列举了不同跳转指令。jmp指令是无条件跳转。它可以是直接跳转，即跳转目标是作为指令的一部分编码的；也可以间接跳转，跳转目标是从寄存器或者内存位置读出的。

汇编语言中，直接跳转直接给出一个标号作为跳转目标（ jmp .L1）。间接跳转的写法是 ”*“后跟操作指示符（jmp *%rax 用寄存器%rax的值作为跳转目标， jmp *(%rax) 用%rax的值作为读地址，从内存读出跳转目标）。

4.4. 跳转指令的编码

  • 跳转指令的两种编码
    • 相对寻址（目的指令的地址和紧跟跳转指令后的地址差），偏移量可以编码为 1、2、4字节
    • 绝对寻址，用4个字节指定目标。

以下是一个相对寻址示例： 

汇编代码：
  mov %rdx,%rax
  jmp .L2
.L3:
  sarq %rax
.L2:
  testq %rax,%rax
  jg    .L3
  rep;  ret

反汇编代码：
0: 48 89 f8   mov  %rdi,%rax
3: eb 03      jmp  8<loop+0x8>  //跳转到地址0x8, 0x3+0x5(下一条指令地址)= 0x8
5: 48 d1 f8   sar  %rax
8: 48 85 c0   test %rax,%rax
b: 4f f8      jg   5<loop+0x5>  //跳转到地址0x5, 单字节0xf8为补码表示，十进制为-8，-8+13(0xd下一指令地址) = 5
d: f3 c3      repz retq         //对AMD的优化，可以直接无视rep

链接后的程序反汇编版本：
4004d0: 48 89 f8   mov  %rdi,%rax
4004d3: eb 03      jmp  4004d8<loop+0x8>
4004d5: 48 d1 f8   sar  %rax
4004d8: 48 85 c0   test %rax,%rax
4004db: 4f f8      jg   4004d5<loop+0x5>
4004dd: f3 c3      repz retq
//上面链接后，跳转目标的编码没有变，通过使用相对偏移，使得指令编码比较简洁（2字节），并且目标代码可以不做改变就能移到内存的不同位置。

4.6. 用条件传送来实现条件分支

处理器通过使用 流水线（pipelining） 来提高性能，在流水线中一条指令的处理需要经过一系列的阶段，每个阶段执行所需操作的一小部分（例如从内存读取指令、确定指令类型、从内存读数据、执行算术运算、向内存写数据，以及更新程序计数器）。这种方法通过重叠连续指令的步骤获得高性能，例如，在取一条指令的同时，执行它前面一条指令的算术运算。这就需要提前确定要执行的指令序列。

4.7. 循环

4.7.1. do_while 循环

/** C代码*/
long fact_do(long n)
{
        long result = 1;
        do {
                result *= n;
                n = n - 1;
        } while (n > 1);
        return result;
}

/** 等价的goto版本 */
long fact_do_goto(long n)
{
        long result = 1;
loop:
        result *= n;
        n = n - 1;
        if (n > 1)
                goto loop;
        reteturn result;
}
/** 汇编代码*/
fact_do:
      mov    $1,%eax
.L2:
      imulq  $rdi,%rax
      subq   $1,%rdi
      cmpq   $1,%rdi
      jg     .L2
      rep; ret

4.7.2. while 循环

4.7.2.1. 方法一：跳转到中间

首先执行一个无田间跳转，跳到循环结尾处的测试，以此来执行初始的测试。 

/** C代码*/
long fact_while(long n)
{
        long result = 1;
        while (n > 1) {
                result *= n;
                n = n-1;
        }
        return result;
}

/** 等价goto版本*/
long fact_whilt_jm_goto(long n)
{
        long result = 1;
        goto test;
loop:
        result *= n;
        n = n -1 ;
test:
        if (n > 1) {
                goto loop;
        }
        return result;
}

/** 汇编代码*/
fact_while:
        movl   $1,%eax
        jmp    .L5
.L6
        imulq  %rdi,%rax
        subq   $1,%rdi
.L5
        cmpq   $1,%rdi
        jg     .L6
        rep; ret
4.7.2.2. 方法二：guarded-do方法

首先用条件分支，如果初始条件不成立，就跳过循环，把代码变换为do-while循环。

4.8. switch语句

swtich语句通过跳转表结构实现。跳转表是个数组，表项i是一个代码段地址，代码段对应当索引值等于i时应当采取的动作。GCC在根据switch case的数量，以及值的稀疏程度来翻译switch case语句。case数量较多（例如4个以上），并且值的跨度范围较小时，就会使用跳转表。

6.2. 变长数组

历史上，C语言多维数组需要在编译时就能确定其大小。在ISO C99中引入了新功能，允许数组的维度是表达式，在数组被分配的时候才计算大小。

变长数组的声明： 

int A[expr1][expr2];

上面的数组A既可以作为局部变量，也可以作为函数参数，然后在遇到这个声明时，通过对表达式expr1和expr2求值，确定数组的维度。例如：要访问 n*n 数组的元素i、j，可以写一个如下函数： 

int var_ele(long n, int A[n][n], long i, long j) {
        return A[i][j];
}

注意，参数n必须在参数 A[n][n] 之前，这样函数就可以在遇到这个数组时，计算数组的维度。

以上函数（var_ele）产生的汇编如下 

var_ele:
      imulq  %rdx,%rdi          计算 n*i
      leaq   (%rsi,%rdi,4),%rax   计算地址  x_{A}+4(n*i)
      movl   (%rax,%rcx,4),%eax   读取内存 M[x_{A}+4(n*i)+4j] 的值
      ret

7. 机器程序中将控制和数据结合

7.1. 理解指针

  • 每个指针都对应一个类型，类型表明指针指向的是哪一类对象
  • 每个指针都有一个值，代表指定类型对象的地址。
  • 指针可以用‘&’运算符创建。该运算符可以应用到任何C表达式的左值上。
  • ‘*’操作符用于间接引用指针
  • 数组与指针紧密联系
  • 将指针从一种类型强制转换为另一种类型，只改变其类型，不会改变值。强制类型转换会改变指针运算的伸缩（指针步长）。
  • 指针可以指向函数–函数指针。

7.2. 对抗缓冲区溢出

  • 栈随机化，在程序开始时，在栈上分配一段随机大小的空间。
  • 栈破坏检测，在栈帧中任何局部缓冲区和栈状态之间存储一个特殊值（canary），canary值是有程序每次运行随机产生，在恢复寄存器状态返回函数前，检查canary值是否被修改覆盖，有，则异常终止。可以通过 “-fno-stack-protector” 选项关闭该机制。
  • 限制可执行代码区域（页的可执行标记）

7.3. 变长栈帧

有些函数，在编译时可以确定栈的大小，即可以确定为栈帧分配多少空间。但是有些函数的局部存储是变长的，例如在栈上分配空间。

遇到变长栈帧时，通过 %rbp 寄存器保存栈基址（base pointer），进入函数前首先将 %rbp压栈，之后执行 movq %rsp,%rbp，保存栈基址。

函数返回前调用 leave指令，该指令等价于如下命令： 

movq %rbp,%rsp        从rbp恢复rsp
popq %rbp             恢复rbp的值
