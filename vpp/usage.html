1. 网卡配置

1.1. dpdk插件

1.1.1. igb_uio 驱动加载

首先 clone dpdk-kmods 项目, 编译 igb_uio 驱动.

  • 编译 

    $ git clone git://dpdk.org/dpdk-kmods
$ cd dpdk-kmods/linux/igb_uio/
$ make    # 需要安装相应的内核头文件

  • 加载 

    $ sudo modprobe uio
$ sudo insmod igb_uio.ko

  • 绑定网卡

    利用 dpdk 源码中的脚本 dpdk-devbind.py 进行网卡绑定. dpdk 项目地址: git://dpdk.org/dpdk 

    $ dpdk-src/usertools/dpdk-devbind.py --status  # 查看网卡的pci号
$ dpdk-src/usertools/dpdk-devbind.py --bind=igb_uio  xxxx:xx:xx.x # 替换位上面查到的pci

1.1.2. vpp配置

修改vpp配置文件的dpdk配置块.

dpdk {
  dev xxxx:xx:xx.x {
        # 名称随意命名
        name enp1s0f0
      }
}

2. LDP

2.1. vcl配置

vcl {
  heapsize 64M
  rx-fifo-size 4000000
  tx-fifo-size 4000000
  api-socket-name /tmp/vpp-api.sock
}

$ VCL_CFG=/path/to/vcl.conf
$ LDP_PATH=/path/to/vpp/build-root/build-vpp[_debug]-native/vpp/lib/libvcl_ldpreload.so
$ sudo LD_PRELOAD=$LDP_PATH VCL_CONFIG=$VCL_CFG nginx -c nginx.conf

3. flowprobe

3.1. 功能

生成ipfix的流记录

3.2. cli

DBGvpp# flowprobe params record l3 active 20 passive 120   # 记录ip层流量信息, 每隔 20s 导出一次, 如果长时间处于非活跃时, 120s 超时后导出一次.
DBGvpp# flowprobe feature add-del  网卡名称  # 在网卡上开启ipfix流记录功能

发送流量信息到收集器, 目的地址 192.168.0.2 端口 4739 源地址 192.168.0.1 发送流量模板间隔20s mtu 1500 

DBGvpp# set ipfix exporter collector 192.168.0.2 port 4739 src 192.168.0.1 template-interval 20 path-mtu 1500
